Tests for Significantly Different Distributions in the STORM/Telomere Project¶

The major results of the telomere project are probability distributions for measuring a given radius of gyration of a telomere. After knockdown of any particular shelterin protein, we would like to know if the size distribution has changed relative to the control. This means we would like to answer the following question: do two sets of samples have different probability distributions?

Testing the distributions for normality¶

Traditionally, a two sample t-test (or one of its variations) is used to determine whether two sample means are significantly different. To perform the two sample t-test, the following requirements should be satisifed: 1. Both populations from which the samples were drawn should be normally distributed. 2. The samples should be independent. Let's look at the box plots that I've already generated for the wild type distributions for Hela L and S to see if they're normal or not.

If the distributions were normal, then these two should lie at the same value for the radius of gyration, but they do not. Therefore, the distributions are probably not normally distributed, but let's double check.

These are incredibly small p-values, so we might at first think that the distributions are likely not normally distributed.

However, there is a very big problem with this conclusion. For large sample sizes, most tests of significance will reject the null hypothesis. This means that small deviations from a perfect Gaussian will be preferentially weighted in the normality test when N is large, leading us to extremely small p-values. Here, a normality test should not be thought of as way to determine whether we can use significance tests that require normality; instead, they only tell us whether there's evidence to say that the distribution is not normal.

Due to these complications a different way to compare two distributions might be required for this project.

See this discussion on normality testing and large sample sizes and this discuss on significance testing with large samples sizes for more information.

The two sample t-test¶

As I discussed above these distributions fail normality tests, but normality of the distributions is a requirement for performing a valid two sample t-test.

On the other hand, the two sample t-test is robust against non-normal distributions for large sample sizes, so let's try running it anyway on these distributions.

Note: strictly speaking, we will use Welch's t-test because the variances of the underlying distributions cannot be assumed to be equal. This is set by the equal_var = False argument.

In [5]: 
from scipy import stats
_, pValue = stats.ttest_ind(control, treatment, equal_var = False)
print("The p-value for Welch's two sample t-test is: %E" % pValue)

The p-value for Welch's two sample t-test is: 2.586018E-08

Here we have a p-value of almost 300 millionths. The test suggests that the mean values really are different, but given the problem with large sample sizes, is this true?

The Mann-Whitney-Wilcoxon rank-sum test¶

The Mann-Whitney-Wilcoxon rank-sum test is a non-parametric test for the null hypothesis that two samples are drawn from the same distribution. The alternative is that measurements in one sample are more likely to be larger than measurements in the other. This non-parametric test does not require normality; indeed, it has very few assumptions, so let's see what it has to say.

In [6]: 
_, pValue = stats.ranksums(control, treatment)
print("The p-value for the Wilcoxon rank-sum test is: %E." % pValue)

The p-value for the Wilcoxon rank-sum test is: 1.708182E-06.

The rank-sum test produces a p-value that is about one millionth, so it would also tell us to reject the null hypothesis and consider the two distributions as different. The question still remains as to whether we can trust this result.

Bootstrapping¶

Bootstrapping techniques are ways of inferring the accuracy of sample estimates, like the mean. They work by treating the measured distribution as the unknown population. Estimates of the accuracy of the population are then derived by sampling with replacement from the measured distribution.

Why would we do this? Well, let's think about why the two sample t-test failed. The t statistic for Welch's two sample t-test for samples with different variances is

\(\bar{X}_1\) and \(\bar{X}_2\) are the sample means, \(s_1\) and \(s_2\) are the sample standard deviations, \(n_1\) and \(n_2\) are the number of measurements in each of the two samples. Now, if the sample size gets very large, this has the effect of making the denominator of \(t\) very small, making \(t\) itself large. \(t\) is what is used to determine the probability in the tails of Student's distribution: large \(t\) values have correspondingly low p-values.

Therefore, the reason the p-value is so small in the t-test is because \(n_1\) and \(n_2\) are relatively large.

The t-test essentially assumes our measurements have no uncertainty, so the only error in estimating the mean value is from undersampling. With large sample sizes, the error from undersampling is small, so the error in the mean estimate should be small. But our measurements have large sources of uncertainty associated with them, and these are not taken into account in the t-test.

To better assess the error in the mean, let's make a bootstrap estimate of its error. We'll first write a bootstrap function that samples from our two distributions with replacement.

Note: See this page for one idea about implementing bootstrap in Python. Also, the Wikipedia page suggests that the size of the number of samples in one resample should equal the size of the original dataset.

In [7]: 
def bootstrap(data, numSamples = None, statistic = 'mean'):
    """Takes an array of measurements and resamples them with replacement.
    
    Arguments
    ---------
    data : array of float
        The data array
    numSamples : int (optional)
        How many times the distribution will be resampled.
        
    Returns
    -------
    bsStat : float
        The bootstrapped mean value.
    """
    if numSamples is None:
        numSamples = data.size
        
    randInts = np.random.randint(0, high = data.size, size = numSamples)
    resampledData = data[randInts]
    
    if statistic == 'mean':
        bsStat = np.mean(resampledData)
    elif statistic == 'median':
        bsStat = np.median(resampledData)
    try:
        return bsStat
    except:
        print("Error! Was the statistic argument 'mean' or 'median'?")
        raise

Resampling without replacement¶

Bootstrapping involves resampling the measured distributions with replacement. What does resampling without replacement tell us?

One thing to be clear about is that resampling without replacement will always lead to larger estimates for the mean uncertainty. This is because we will have, in general, fewer data points in our resample than in the original sample of measurements.

In [17]: 
def resampleNoReplace(data, numSamples):
    """Takes an array of measurements and resamples them without replacement.
    
    Arguments
    ---------
    data : array of float
        The data array
    numSamples : int
        How many times the distribution will be resampled. This must be smaller than data.size.
        
    Returns
    -------
    rsMean : float
        The resampled mean value.
    """
    np.random.shuffle(data)
    try:
        rsMean = np.mean(data[:numSamples])
    except:
        print("Error: numSamples must be smaller than the size of the input data array.")
        
    return rsMean

At this point we could ask the following question: at what sample size does subsampling without replacement produce mean gyration radii that are not statistically different?

My hypothesis is that the smaller this value for the sample size, the more likely the means are significantly different. When the sample size is equal to the size of the individual distributions, the resampled mean will be the same as the sample means. The resampled means will be equivalent in this case only if the sample means are equivalent.

On the other hand, if the resample size is one, the distributions should be uniformly distributed over the full range of values in the original sample, and there will be significant over lap between the Control and Treatment distributions. In this case, undersampling should be the main source of error in the mean value.

The lower the size of the resamples for a significantly different result, the less the spread in the mean estimates is due to undersampling and the more it reflects the true dispersion of the real, measured sample means.

Therefore, it's not likely that undersampling would cause us to accept the null hypothesis with resamples of size 20. We would likely conclude that the distribution means are different.
